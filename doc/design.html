Spring AI Agent 前端技术方案

React 18 + TypeScript + Vite Tailwind CSS 主题定制 2026-05-10

一项目概述

为 Spring AI Agent 后端（/api/agent/chat 和 /api/agent/stream-chat）构建一个现代化的 React 前端界面，提供智能对话交互能力。

二技术选型

技术 版本 说明
React18.xUI 框架
TypeScript5.x类型安全
Vite5.x构建工具（HMR 快速开发）
Tailwind CSS3.x原子化 CSS，快速定制主题
ahooks3.x常用 React Hooks 工具集
clsx2.x条件 class 拼接
date-fns3.x时间格式化

不使用重量级 UI 组件库（Ant Design / Element Plus），保持包体积和设计自由度。

四API 接口对接

4.1 阻塞式对话

POST /api/agent/chat
Content-Type: application/json

{
  "query": "用户问题",
  "sessionId": "uuid-string",
  "userNo": "user-001"
}

{
  "code": 200,
  "data": {
    "answer": "AI 回复内容",
    "sessionId": "uuid-string"
  }
}

4.2 流式对话

POST /api/agent/stream-chat
Content-Type: application/json
Accept: text/event-stream

{
  "query": "用户问题",
  "sessionId": "uuid-string",
  "userNo": "user-001"
}

Response:
X-Session-Id: uuid-string
Body: text/event-stream (data:回复片段...)

前端使用 fetch + ReadableStream 读取并逐步追加渲染。

4.3 健康检查

GET /api/agent/health
Response: { "code": 200, "data": "Agent service is running" }

五核心数据流

5.1 状态管理（React Hook 方案）

不使用 Redux / Zustand 等第三方状态库，采用 React Hooks + Context 方案：

SessionContext
├── sessionId: string           // 当前会话 ID
├── userNo: string              // 用户标识
└── newSession: () => void      // 新建会话

useChat Hook
├── messages: Message[]          // 消息列表
├── isProcessing: boolean        // 是否正在处理
├── mode: 'blocking' | 'stream'  // 当前模式
├── error: string | null         // 错误信息
├── sendMessage: (query) => void // 发送消息
├── stopStream: () => void       // 停止流式
├── setMode: (mode) => void      // 切换模式
└── clearMessages: () => void    // 清空消息

5.2 数据流图

用户输入 → sendMessage(query) │ ├── mode === 'blocking' ──→ POST /api/agent/chat │ │ │ └──→ 收到完整响应 → 追加消息 │ └── mode === 'stream' ──→ POST /api/agent/stream-chat │ ├── 创建空 AI 消息占位 ├── fetch ReadableStream 读取 chunks ├── 逐块追加到 AI 消息 content ├── 自动滚动到底部 └── 流结束 → 标记 complete

5.3 消息模型

interface Message {
  id: string;                    // 唯一标识
  role: 'user' | 'assistant';    // 发送者
  content: string;               // 内容
  timestamp: number;             // 时间戳
  status: 'sending' | 'streaming' | 'complete' | 'error';
}

6.3 交互细节

  1. 流式模式：发送后按钮变为"停止"（红色），点击可中断 SSE 连接
  2. 阻塞模式：发送后按钮显示加载旋转，不可中断
  3. 自动滚动：新消息到达时自动滚到底部；用户向上滚动查看历史时暂停
  4. 快捷键：Enter 发送，Shift+Enter 换行
  5. 输入高度：textarea 自动增高，最多 6 行，超出后出现滚动条
  6. 空状态：无消息时显示欢迎语和示例问题
  7. 错误处理：消息气泡内显示错误状态，可点击重试

八CORS 方案

后端 springai-agent 运行在 localhost:8082，前端开发在 localhost:5173（Vite 默认），存在跨域问题。

开发环境：Vite proxy 配置

// vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8082',
        changeOrigin: true,
      },
    },
  },
});

生产环境

Nginx 同域代理或后端加 @CrossOrigin

九性能考虑

  • 流式渲染使用 requestAnimationFrame 节流，避免高频更新卡顿
  • 消息列表虚拟化（超过 100 条时开启）
  • 代码块使用懒加载的代码高亮组件
  • API 客户端复用 AbortController 实现请求取消
